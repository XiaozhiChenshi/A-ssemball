# 序章特效改造工作步骤

## 目标
为 `intro_interactive`（序章）场景补充更真实的视角抖动、修正光照、增加“摩尔纹失真-随机噪点”后处理特效，并保持现有交互流程不回归。

## 现状基线（已确认）
- 场景文件：`scenes/intro_interactive.tscn`
- 脚本文件：`scripts/intro_interactive.gd`
- 当前已有：
  - 左右循环式屏幕抖动（`_play_screen_shake`）
  - 黑角（vignette）`ColorRect + canvas_item shader`
  - 序章 3D 内容在 `SubViewport` 内渲染

## 实施顺序

### 步骤 1：相机抖动随机化（幅度 + 速度）
1. 在 `scripts/intro_interactive.gd` 中扩展抖动参数：
   - 基础幅度
   - 幅度随机区间（建议用倍率或 min/max）
   - 速度随机区间（每半步或每循环随机）
2. 改造 `_play_screen_shake`：
   - 保留“往复震动 + 衰减”主结构
   - 将每一步的位移幅度和步进时长做随机采样
   - 避免过度突兀：加入上下限钳制，保证总时长接近原值
3. 保持与现有过渡流程兼容：
   - 仍在点击白球后触发
   - 不破坏后续黑场与竖线转场

验收标准：
- 抖动不再机械左右等幅等速；
- 每次触发抖动节奏有细微差异；
- 不出现明显卡顿、瞬移或长时间停顿。

---

### 步骤 2：取消序章模型字体发光 + 场景内添加两盏扁平自然光
1. 先定位“序章模型字体”节点及其材质来源（若在 `intro_interactive.tscn` 内，直接改该场景；若为外部实例，定位其源场景）。
2. 去除字体发光表现：
   - 优先关闭/降低材质 `emission`（或替换为无自发光材质）
   - 保证文字仍可见但不自发亮
3. 在序章 3D 场景中**直接添加**两盏高位自然光（非仅代码动态创建）：
   - 在 `scenes/intro_interactive.tscn` 的 `WorldRoot` 下新增 2 个光源节点
   - 位置在较高处，方向形成主辅光，整体偏“自然、柔和、扁平”
   - 调整颜色、能量、阴影开关，避免过曝与死黑

验收标准：
- 字体不再发光刺眼；
- 场景层次更清晰，地面/目标物/字体可读性提升；
- 灯光节点在场景文件中可见，可在编辑器直接调参。

---

### 步骤 3：新增“摩尔纹失真-随机噪点”后处理 shader
1. 在 `scenes/intro_interactive.tscn` 新增后处理层（`ColorRect`）及 `ShaderMaterial`。
2. 编写 `canvas_item` shader，实现两类效果并可调：
   - 摩尔纹/扫描干涉感（基于屏幕 UV 的条纹干涉）
   - 随机噪点（时间变化的颗粒噪声）
3. 通过参数控制强度，保证不盖过现有黑角：
   - 支持总强度、摩尔纹强度、噪点强度、时间速度
   - 默认值先“轻度可见”
4. 确认叠加次序：
   - 与 `VignetteOverlay` 的层级关系可控（通常噪点在上、黑角保留）

验收标准：
- 序章画面可见轻微摩尔纹与随机噪点；
- 效果会随时间动态变化，不是静态贴图感；
- 不影响点击白球检测与转场逻辑。

## 联调与回归检查
1. 进入序章，完整走一遍：按住 `W` 前进 -> 到终点 -> 点击白球 -> 抖动 -> 黑场 -> 竖线转场。
2. 检查项：
   - 抖动随机性稳定可控
   - 灯光改善后无明显曝光问题
   - 新 shader 不引发闪烁异常或性能尖峰
3. 参数微调并固化默认值（保留导出参数便于后续继续调）。
